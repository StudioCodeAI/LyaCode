# Lya Code Brand Assets

Lya Code is a Studio CodeAI product. Its visual identity must be independent from older Studio CodeAI projects while keeping a recognizable family feeling.

## Official Mark System

The Lya Code mark system combines three ideas:

- Atomic intelligence: the central gold nucleus and orbit system.
- Cloud/runtime presence: the cyan living orbit around the core.
- Terminal workflow: compact recognition for CLI, docs, extension, and release surfaces.

Official files:

- `assets/brand/lyacode-logo.svg` - clean main logo for CLI, favicon, npm, and compact identity.
- `assets/brand/lyacode-app-icon.svg` - framed app/extension icon.
- `assets/brand/lyacode-favicon-thinking.svg` - animated thinking/startup favicon.
- `assets/brand/lyacode-logo-1024.png`
- `assets/brand/lyacode-logo-512.png`
- `assets/brand/lyacode-app-icon-1024.png`
- `assets/brand/lyacode-app-icon-512.png`
- `docs/assets/lyacode-readme-hero.png`
- `web/public/lyacode-logo.svg`
- `web/public/lyacode-app-icon.svg`
- `web/public/favicon-thinking.svg`
- `web/public/lyacode.png`
- `vscode-extension/lyacode-vscode/media/lyacode.svg`

Usage:

- Main CLI logo and favicon: use the clean logo.
- App stores, extension icons, social cards, and README hero: use the framed app icon.
- Startup, waiting, thinking, or loading browser states: use the animated thinking favicon.

## Legacy Reference Assets

The asset folder `E:\GitHub\LyaCode\assets` contains visuals from another Studio CodeAI project. Those files can guide color, energy, and family language, but they must not be copied into Lya Code unchanged.

The first small alternative mark from `lya-brand.jpg` is the approved direction for the CLI icon. The Lya Code production icons are cleaned vector derivatives of that direction, not direct copies of the older project export.

Do not reuse these older marks directly:

- Circular `Lya Studio Code` logo rings.
- Existing app icon exports from the previous project.
- Brand boards or screenshots that identify another product.

If a legacy image is needed for Lya Code marketing, create an edited derivative that removes old project naming and changes the composition enough to become a Lya Code-specific asset.

## Naming

- Product text: `Lya Code`
- CLI and package identity: `lyacode`
- Package scope: `@studiocodeai/lyacode`
- Environment variables: `LYACODE_*`
- Code identifiers: `lyaCloud` or `LyaCode`

`Claude Code` may remain only where it refers to provider compatibility, SDK references, or upstream technical behavior. It is not the Lya Code product identity.
